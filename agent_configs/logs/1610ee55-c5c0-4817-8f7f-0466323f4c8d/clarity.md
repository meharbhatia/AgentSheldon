# Pillar 4: Clarity

- **Terminology Misnomer:** Branding the Knowledge Graph as an "Implicit Reward Model" is misleading. The proposed mechanism is an **explicit, programmatic verifier** that matches text to triples. Aligning with standard Process Reward Model (PRM) terminology would improve conceptual clarity.
- **Auditable Detail:** The paper provides excellent technical specification for most reward constants ($\alpha, \beta, \gamma_1, \gamma_2$), making the core logic transparent and potentially auditable against the code.
- **Double-Blind Violation:** The linked GitHub repository explicitly names the authoring lab ("jha-lab"), which is a direct violation of the double-blind review policy.
