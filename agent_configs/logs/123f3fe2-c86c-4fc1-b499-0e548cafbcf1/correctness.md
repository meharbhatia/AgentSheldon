# Pillar 1: Correctness

- **Algorithmic Complexity Error:** There is a significant error in the complexity claims in Section 3.6. The authors assert that batch processing reduces memory complexity to $O(L)$. However, Algorithm 2 (Backtracking) requires the full DP table $g$ of size $(2L+1) \times (K+1)$, where the latency budget $K$ scales linearly with context length $n$ and layer count $L$. Thus, the memory overhead is at least $O(n L)$. Furthermore, the total theoretical work remains $O(n^2 L)$ because each of the $O(n L)$ DP states involves a forward pass with $O(n)$ Attention complexity.
- **Operating-Point Theory-Practice Gap:** While Lemma 4.1 (Eq 10) provides a mathematically sound sufficient condition for greedy token preservation, the required cosine similarity threshold is extremely restrictive. For typical LLM head dimensions and logit margins, the guarantee requires $\cos(x, x') \approx 1 - 10^{-6}$. In contrast, the framework operates at $\tau = 0.5$, meaning the theoretical guarantee does not apply to the vast majority of practical drafting scenarios.
- **Sub-layer Atomicity Paradox:** By decoupling Attention and MLP layers, KnapSpec assumes the Transformer block is a separable update sequence. However, in modern architectures, the MLP is calibrated to process the attentional update. Independent skipping induces a distributional shift in the residual stream that likely forces the draft model into an out-of-distribution regime, making per-layer cosine similarity a potentially biased proxy for final acceptance rate.
- **Heuristic Nature of DP:** The DP recurrence (Eq 8) is locally greedy, selecting the candidate that maximizes similarity to the layer-$i$ target reference. Since the residual stream drifts cumulatively, maximizing per-layer cosine does not guarantee global optimality for the end-of-network similarity or the acceptance rate $\alpha_S$.
