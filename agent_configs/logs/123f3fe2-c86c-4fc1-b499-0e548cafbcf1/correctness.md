### Pillar 1: Correctness

- **Theoretical-Practical Gap in Lemma 4.1**: The paper provides Lemma 4.1 to justify using cosine similarity as a proxy for the token acceptance rate. However, the condition required for the guarantee (Eq. 10) depends on the margin \xi(x) and the embedding norm. In practice, for a large vocabulary, \xi(x) is often very small, requiring a cosine similarity extremely close to 1 (e.g., >0.99). The paper's actual operating point uses a pruning threshold of \tau=0.5 (Section 5.2), which is vastly lower than what the lemma would likely require for a guarantee. This indicates that the theoretical link is largely decorative and does not rigorously support the empirical configuration.
- **Flawed Complexity Claims**: In Section 3.6, the paper claims that batch processing reduces the runtime complexity to O(nL) and memory complexity to O(L). This is mathematically incorrect. The DP table g has (2L+1) x (K+1) entries, where K is the total weight budget (proportional to L). Each entry is a hidden state of size r x d. Thus, memory complexity is O(L^2 rd). For runtime, even if K forward passes are batched, the total compute per layer is K times the cost of a single pass. The total compute is thus O(n L^2), not O(nL). Claiming the same complexity as standard decoding by assuming "batching" ignores the actual resource consumption and hardware constraints.
- **Residual Drift in Sub-layer Decoupling**: The paper decouples Attention and MLP layers for independent skipping. In standard Transformer architectures (like Llama and Qwen mentioned in the paper), these sub-layers are connected residually, where the MLP input is typically the sum of the layer input and the Attention output (h_MLP = h_in + Attn(LN(h_in))). If the Attention sub-layer is skipped but the MLP is kept, the MLP receives h_in as input, which is a significant distribution shift from what it was trained on. This "non-atomic" skipping likely causes severe representation drift that is not fully captured by the cosine similarity proxy.
- **Suspicious Baseline Results**: In Table 1, the "DEL" baseline is reported to have 0% acceptance rate for multiple configurations. While DEL is an early-exit method, 0% acceptance is remarkably low for a state-of-the-art baseline, suggesting it may have been evaluated without proper hyperparameter tuning or in a configuration that intentionally disadvantages it.
